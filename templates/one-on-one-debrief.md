## Propagation list

The debrief is done when these are filed, not when the file is written.

- [ ] Add / update contact row in your cross-company tracker (e.g. `connections.md`, Notion CRM, Airtable)
- [ ] If outreach context exists, append last-touch note to relevant `outreach-log.md` or equivalent
- [ ] File any **adopt** signal as a backlog item with link back to this debrief
- [ ] File any **consider** signal as a workflow insight or friction log row
- [ ] Calendar reminder for next-touch (if cadence was specific)
- [ ] Send any promised follow-up artifact (link / intro / answer to open question) within 48h while the thread is warm

If you don't have downstream homes for these, the relationship dies in this file. Build the homes before running another 1-on-1.
